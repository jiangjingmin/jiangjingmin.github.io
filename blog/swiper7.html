# 关于 antd pro 使用 swiper7 的问题

# 背景

有个需求要搞个轮播图呀，很复杂的，手写不太好实现，所以就打算使用 swiper 来解决。

# 问题描述

好家伙，引入就报错了，真是一点面子都不给。

load component failed Error: Module "./swiper/react" does not exist in container.

Uncaught (in promise) Error: Module "./swiper/react" does not exist in container.
while loading "./swiper/react" from webpack/container/reference/mf

# 原因分析

我觉得是是不兼容问题。然后搜了半天，看到 swiper version 7 only works if you have pure ESM 。 牛呀，只支持 pure ESM ，这么重要的事情感觉在官方文档没看到呢。

# 解决方案

# 方案一

降级使用 swiper6.8.4 版本。 swiper6 的官网地址 https://swiper6.vercel.app/react (opens new window)

npm install swiper@6
# or
yarn add swiper@6

// Import Swiper React components
import SwiperCore, { Autoplay } from 'swiper';
import { Swiper, SwiperSlide } from 'swiper/react';

// Import Swiper styles
import 'swiper/swiper.less';

// install Swiper modules
SwiperCore.use([Autoplay]);

// install Swiper modules
SwiperCore.use([Lazy,Pagination,Navigation]);

<Swiper
  className={styles.alarmPanel}
  direction={'vertical'}
  // allowTouchMove={false}
  autoplay={
    swiperList.length > slidesPerViewNum
      ? {
          delay: 0,
          disableOnInteraction: false,
        }
      : false
  }
  speed={1000}
  loop={swiperList.length > slidesPerViewNum}
  slidesPerView={slidesPerViewNum}
>
  {swiperList.map((item: swiperItem, index: number) => {
    return (
      <SwiperSlide key={item.title + index} className={styles.alarmSlide}>
        {item.title}
      </SwiperSlide>
    );
  })}
</Swiper>

有时候会加载css或者其他的报错呀，删掉 .umi 文件夹，重新启动就可以啦！

# 方案二

继续使用 swiper7 ，那么你需要看看这个文档将自己的项目改造为 pure ESM(纯ESModule) ，感觉很麻烦而且可能会产生更多问题呢，所以肯定就不用这个了。如果你想用可以参考这篇来进行改造。 如何将项目改造为pure ESM
